# Путеводитель по кодовой базе

Этот проект — очень компактный статический лендинг без сборщика и без backend.

## Структура

- `index.html` — разметка страницы: hero-блок, секции «О проекте» и «Контакты», форма и контейнер для результата.
- `styles.css` — базовые стили, адаптивность и стили формы.
- `script.js` — клиентская логика формы: перехват submit, простая валидация, сообщение пользователю.

## Как всё работает вместе

1. Браузер загружает `index.html`.
2. Через `<link>` подтягиваются стили из `styles.css`.
3. Внизу документа подключается `script.js`, который после `DOMContentLoaded` вешает обработчик на форму.
4. При отправке формы данные не уходят на сервер: показывается локальный статус в `#result`.

## Что важно знать новичку

- Проект можно запускать просто открытием `index.html` в браузере.
- `required` и `type="email"` уже дают часть нативной HTML-валидации.
- Текущая JS-валидация проверяет только непустые `name` и `email`.
- Отправка пока моковая: для реальной интеграции нужно добавить `fetch` к API.

## Что изучить дальше

1. **Семантическая HTML-разметка и доступность (a11y)**: `label` для полей, фокус-стили, ARIA-паттерны.
2. **Улучшение валидации**: проверки формата email, длины сообщений, сообщения об ошибках у конкретных полей.
3. **Интеграция с backend**: `fetch`, обработка ошибок сети, состояния загрузки, защита от повторных отправок.
4. **Организация CSS**: переменные (`:root`), методологии (BEM), вынесение дизайн-токенов.
5. **Тестирование**: базовые unit/e2e-тесты формы.

## Быстрые команды для разработки

```bash
# Вариант 1: просто открыть файл
xdg-open index.html

# Вариант 2: поднять локальный сервер на Python
python3 -m http.server 8000
# затем открыть http://localhost:8000
```
